# Current capability status

This audit records what the current application can do, which paths still need
work, and where each conclusion comes from.

It reflects commit `d4227fc` plus the pre-existing worktree changes present at
the audit preflight. A **user entry point** is a control or page a person can
reach in the current browser interface; the existence of a Flask route alone
does not make a capability user-facing.

## Status labels

- `supported` — wired through a current user interface to its backend
  implementation.
- `experimental` — user-facing, but dependent on optional or external systems,
  or not yet covered well enough to treat as a dependable primary path.
- `backend-only` — implemented behind routes or pipeline functions but absent
  from the registered browser workflow.
- `blocked` — an interface advertises the capability, but the current wiring
  cannot complete it.
- `historical` — retained only as earlier behavior or reference material.
